## 🏛️ 架构：两府三司制
02769D 的架构设计参考了北宋“两府三司”的权力制衡思想，将推理过程拆解为三个职能独立、相互制约的模块：

| 古代机构 | 现代职能 | 核心职责 |
| :--- | :--- | :--- |
| **西府（枢密院）** | **语义翻译模块** | 掌“兵权”。调用 GLM-5.1，**严格限制**其仅能将自然语言翻译为纯数学算式，**严禁其执行任何计算、推理或算术操作**。 |
| **东府（中书门下）** | **符号计算模块** | 掌“政权”。内嵌 **SFCP 确定性计算引擎**，负责执行所有数学运算，是系统的核心算力来源。 |
| **三司（盐铁、度支、户部）** | **审计与斩杀模块** | 掌“审计”。负责审核算式合规性，**斩杀**所有越界输出（如含变量、语法错误），并自动修正错误的标准答案。 |

**没有任何一个模块能单独完成推理。越权行为会被流程自动阻止。这是 AI 推理的“宪政框架”。**

## ⚙️ 核心逻辑：SFCP六步
定源 → 分裂 → 计算 → 剔除 → 保存 → 映射

**定源**：定义边界约束。不是写死规则，是划定合法空间。
**计算**：模型只做翻译，计算完全交给符号引擎。
**剔除**：边界强制执行。含变量、语法错误直接斩杀。
**保存**：版本化快照。每一次运行都有完整存档。

## 🧠 内核：空间分形计算范式 (SFCP)
02769D 的数学内核是 **空间分形计算范式（Spatial Fractal Computing Paradigm, SFCP）**。它源于一个观察：分形结构的无限复杂性，来自极简边界约束下的无限迭代。

**02769D。不是更强的模型。是让模型变得可靠的流程。**

---

本项目现阶段优先深耕小学数学推理场景，依托 SFCP 确定性计算与两府三司架构，实现零样本下超高准确率与计算稳定性；后续将逐步拓展初中代数、几何及更高阶数学推理能力。
