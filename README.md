## 🧠 底层驱动：SFCP 六步范式

| 步骤 | 职责 | 代码实现 |
|------|------|----------|
| **定源** | 定义边界约束，划定合法空间 | 中央 Prompt |
| **分裂** | 将自然语言拆解为可计算的单元 | 西府（GLM 翻译算式） |
| **计算** | 执行数学运算，得出数值结果 | 东府（SFCPEngine） |
| **剔除** | 斩杀越界数据（变量、语法错误） | 三司（ThreeBureaus） |
| **保存** | 版本化快照，完整存档 | `get_next_version()` |
| **映射** | 将计算结果对应回原题答案格式 | 东府 + 三司修正 |

## 🏛️ 外层架构：两府三司

两府三司是 SFCP 六步在 GSM8K 任务上的拟人化封装，便于工程落地和理解：

| 机构 | 对应 SFCP 步骤 | 职责 |
| :--- | :--- | :--- |
| **中央** | 定源 | 定义边界约束 |
| **西府** | 分裂 | 翻译算式 |
| **东府** | 计算 + 保存 | 执行运算、存档结果 |
| **三司** | 剔除 + 映射 | 审核斩杀、修正答案 |

**运行流程**：定源 → 分裂 → 计算 → 剔除 → 保存 → 映射

## ⚙️ 关键机制

| 机制 | 说明 | 对应 SFCP 步骤 |
|------|------|----------------|
| **强制括号** | Prompt 要求必须使用括号 | 定源 |
| **自动补括号** | 修正模型输出中的括号缺失 | 分裂 |
| **变量斩杀** | 拦截含变量的表达式 | 剔除 |
| **偏差修正** | 偏差超20%以计算结果为准 | 映射 |
| **版本化存档** | 每次运行结果独立保存 | 保存 |
