## 🏛️ 架构：两府三司制

02769D 的架构设计参考了北宋“两府三司”的权力制衡思想，将推理过程拆解为三个职能独立、相互制约的模块：

| 古代机构 | 现代职能 | 核心职责 |
| :--- | :--- | :--- |
| **西府（枢密院）** | **语义翻译模块** | 调用 GLM-5.1，将自然语言翻译为纯数学算式。**不参与计算**。 |
| **东府（中书门下）** | **符号计算模块** | 内嵌 **SFCP 确定性计算引擎**，执行所有数学运算。 |
| **三司（盐铁、度支、户部）** | **审计与斩杀模块** | 审核算式合规性，斩杀越界输出，自动修正错误答案。 |

**三个模块协同完成推理。这不是一个“模型”，而是一套“模型驱动的确定性系统”。**

## ⚙️ 核心逻辑：SFCP六步

**定源 → 分裂 → 计算 → 剔除 → 保存 → 映射**

| 步骤 | 说明 |
|------|------|
| **定源** | 定义边界约束，划定合法空间 |
| **分裂** | 将复杂问题拆解为可独立计算的单元 |
| **计算** | 模型只做翻译，计算完全交给符号引擎 |
| **剔除** | 边界强制执行，含变量、语法错误直接斩杀 |
| **保存** | 版本化快照，每次运行都有完整存档 |
| **映射** | 将计算结果映射回原始问题结构 |
