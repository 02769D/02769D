⚙️ 核心逻辑：SFCP六步
 
定源 → 分裂 → 计算 → 剔除 → 保存 → 映射
 
定源：定义边界约束。不是写死规则，是划定合法空间。
计算：模型只做翻译，严格禁止模型参与任何计算与推理，计算完全交给符号引擎。
剔除：边界强制执行。含变量、语法错误直接斩杀。
保存：版本化快照。每一次运行都有完整存档。
 
🏛️ 两府三司 完整架构（与源码一一对应）
 
02769D 内核基于 空间分形计算范式（SFCP），采用自研两府三司工程架构：
 
西府（WestMansion）
 
- 承载主体：GLM-5.1
​
- 唯一职责：自然语言题目 → 标准数学算式翻译
​
- 强约束：禁止参与任何数学计算、逻辑推理、算术运算
​
- 配套能力：错题重试翻译、版本化算式存档
 
东府（EastMansion）
 
- 承载主体：SFCP 空间分形符号计算引擎
​
- 唯一职责：接收西府输出的算式，做确定性安全计算
​
- 内置自动补括号、非法字符过滤、变量检测、安全求值
 
三司（ThreeBureaus）
 
- 审核司：校验算式合法性、格式合规性、是否含英文字母变量
​
- 斩杀司：对空算式、过短算式、语法错误、含变量题目直接拦截剔除
​
- 修正司：校验模型标准答案误差，超阈值自动校准修正原题答案
 
🧠 架构定位
 
02769D。不是更强的模型。是让大模型数学推理变得绝对可靠的工程流程。


 
本项目现阶段优先深耕小学数学推理场景，依托 SFCP 确定性计算与两府三司架构，实现零样本下超高准确率与计算稳定性；后续将逐步拓展初中代数、几何及更高阶数学推理能力。
